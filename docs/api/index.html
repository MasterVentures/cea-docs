CEA Orderbook API 1.0.0

This document details the behaviour of the order book WebSocket service, how to implement its methods and the expected results while making WebSocket requests.

Servers

https://dev-api.cealliance.io/your-exchange-id/ https

Development instance

Security:
  • API key:

    All request to the API require an API_KEY header containing the provided apikey

     In: user

Pub taker/orderbook/get

Accepts the following message:

Orderbook Reponse orderbook response

Start a WebSocket connection used by the exchange to receive aggregated orderbook for the specified symbol

Payload
object
symbol
required
string
length >= 6

pair symbol

depth
required
integer
>= 0

order book depth, a depth of 0 means full book.

subscriptionType
required
integer

1 - To subscribe to aggregated orderbook for the specified symbol. 2 - To unsubscribe or request once

Enum: 1 2

Additional properties are allowed.

Examples

orderbook response
Payload 
{
  "symbol": "ETHBTC",
  "depth": 6,
  "subscriptionType": 1
}

object

required
string

OrderID is a unique alphanumemric identifier for the order.

symbol
required
string
length >= 6

order symbol

orderSide
required
integer

Enum: 1 2
orderType
required
integer

1 - Market Order 2 - Limit Order 3 - Stop Order

Enum: 1 2 3

string
float

Decimal quantity of the order.

price
string
float

Required for Limit or Stop orders.

minQty
string
float

Minimum fill size for the oder.

timeStamp
required

Epoc time of the order

priceDeviation

object

string

Sub taker/order/set/update

Accepts the following message:

Get Order Update Get order update

This channel allows an exchange to submit update or cancel request for active orders.

Payload orderUpdate
object
orderID
required
string

OrderID is a unique alphanumemric identifier for the order.

status
required
integer

1 - Order Created. 2 - Order Rejected. 3 - Order Filled. 4 - Order Partially Filled. 5 - Order Cancelled.

Enum: 1 2 3 4 5
executionID
required
string

unique alphanumeric id

fillQty
string
float

for status 3 or 4, the size of the fill

fillPrice
string

price used for this fill

missingQty
string

missing quantity from the order

execQty
string

executed quantity across all fills

avgPrice
string

Average price for the executed fills

settleDate
string
yyyymmdd

date of the fill, don't use hyphens backslash or any other separator

transactTime
integer

timestamp of the update

Additional properties are allowed.

Examples

Get order update
Payload 
{
  "orderID": "49a58c4c-7b43",
  "status": 1,
  "executionID": "2s3s56",
  "fillQty": "50.00",
  "fillPrice": "0.2521",
  "missingQty": "25.50",
  "execQty": "30.00",
  "avgPrice": "0.36",
  "settleDate": "20091110",
  "transactTime": 1614682480
}
